# Инструкция по деплою на Render.com

## Настройка сервиса

1. **Создайте новый Web Service** на Render.com
2. **Подключите ваш репозиторий**
3. **Настройте следующие параметры:**

### Build Command:
```
pip install -r requirements.txt
```

### Start Command:
```
gunicorn --bind 0.0.0.0:$PORT app:app
```

### Environment Variables:
- `SECRET_KEY` - сгенерируйте случайный ключ (можно через: `python -c "import secrets; print(secrets.token_hex(32))"`)
- `DATABASE_URL` - если используете PostgreSQL на Render, он будет автоматически

### Root Directory:
Оставьте пустым (или укажите корень проекта)

## Важно:

1. Убедитесь, что **Procfile** находится в корне проекта
2. После первого деплоя БД будет автоматически инициализирована
3. Если используете SQLite, файл БД будет создан автоматически
4. Если используете PostgreSQL, создайте базу данных на Render и используйте переменную `DATABASE_URL`

## Проверка:

После деплоя откройте ваш сайт и проверьте:
- Главная страница загружается
- Можно войти (admin/admin123 или user/user123)
- Новости отображаются
- Архив обращений работает
